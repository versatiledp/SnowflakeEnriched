In this book, we have demonstrated how to create an analytical data platform to provide business value using a Snowflake echo system.

We have briefly explained on differentiating features in the Snowflake and quickly moved to create a raw data layer (staged files) and define schema for staged files in operational data tables. We started our journey by using a real-world dataset to create meaningful insights for the business users.

The book portrays extensive details on how to load the data into the Snowflake from files of various data formats like CSV, JSON, ORC, and Parquet. Snowflake stored procedures along with views are used to manage the complex business logic to transform data into the servicing layer.

We have covered how to handle the structural changes in the incoming data files and manipulate the ETL for handling those critical changes. The book has covered various methods on how to merge data from operational data tables into the servicing layer tables. We have explained how to load data from a single file into multiple entities with a different set of attributes and business logic.

The workflow management process is critical for the daily data load in any organization managing data pipelines. Simple elegant and a practical solution to workflow management using Snowflake is explained in this book.

The most important aspect of a data platform is insights. Insights come from data visualizations, reports, predictive analytics. We have built the servicing layer tables and touched on data visualization tools used for presenting the data to business users. Special attention is given to build business use-cases for the reader to visualize the real-world scenarios.

The entire book is written using the best practices on loading and transforming data along with source code. We hope this book servers the reader as an end-to-end coding experience for an aspiring database community to understand Snowflake.
